1 · Spielablauf, Host, Lobby & Timer

Kernspielfluss von Start bis Auflösung. Mehrere Muss-Funktionen sind hier noch offen.

Server als maßgebliche Zeitquelle (serverseitiger Timer)
Der Timer läuft ausschließlich im Browser-JS (game.php ~Z.402-437). Es gibt keinen Frage-Startzeitpunkt in der DB und beim Antworteingang wird nicht geprüft, ob die Antwort rechtzeitig kam. Der Server muss entscheiden, ob eine Antwort gültig ist. → Frage-Start-Timestamp in der DB ergänzen und in next_question.php prüfen.
offenMussLastenheft §13.4 / §26.3
Frage endet zuverlässig bei Timerablauf
„Alle haben geantwortet" funktioniert. Das Timeout-Ende ist aber kaputt: Das Formular sendet das Feld timeout=1 nie, daher ist $isTimeout im Server immer false. Ein Spieler mit geschlossenem Tab blockiert das Weiterschalten dauerhaft (answeredPlayers ≥ totalPlayers wird nie erreicht). → serverseitige Frist nötig (siehe Punkt oben).
teilweiseMussLastenheft §13.3
Antwort bis Frageende änderbar – „letzte gespeicherte zählt"
Nach dem Bestätigen wird der Spieler in den Wartezustand versetzt; INSERT IGNORE auf den Unique-Key (next_question.php Z.102) verhindert ein Überschreiben. Es zählt damit die erste Abgabe, nicht die letzte – und Ändern ist nicht möglich.
teilweiseMussLastenheft §13.1
Host kann einzelne Teilnehmer entfernen (Kick)
Es gibt keine Kick-Funktion – kein Button, kein Endpoint. Damit auch keine Sperre, dass entfernte Teilnehmer nicht weiter antworten. Ist explizite Muss-Funktion.
offenMussLastenheft §9.5 / §29
Optionales Bild zur Frage anzeigen (im Spiel)
game.php rendert nur den Fragetext (Z.222), kein Bild. Es gibt keinen Medien-Bezug zur Frage in der Spielanzeige, obwohl Fragen ein Bild haben können.
offenMussLastenheft §12.1 / §29
Host-Token (geheim, getrennt vom Teilnahme-Code) + Wiedereinstieg
Es existiert kein Host-Token. Host-Steuerung hängt nur an der PHP-Session (quiz_setup.lobby_id); Cookie-Lifetime ist 0 → nach Browser-Schließen ist die Rückkehr in die Host-Rolle unmöglich. Lastenheft/Abweichung verlangen einen separaten, gehashten Token (games.host_token_hash ist im Schema vorhanden, wird aber nicht genutzt).
offenMussLastenheft §9.4 / §29 · Abw. §4
Zu kleiner Fragenpool verhindert Spielstart (mit Fehlermeldung)
Keine Prüfung der Pool-Größe. Ist der Pool kleiner als die gewünschte Fragenanzahl, schneidet array_slice (host_lobby.php ~Z.89) stillschweigend ab – kein verhinderter Start, keine verständliche Fehlermeldung wie gefordert.
teilweiseMussLastenheft §11.3 / §32
Host sieht Antwort-Fortschritt „X / Y haben geantwortet"
Kein Live-Zähler während laufender Frage. Der Host (Dozentenmodus) sieht nur „Warte auf die Antworten…", keine Anzahl. get_lobby_status.php liefert keine Antwort-Anzahl.
offenSollLastenheft §9.5
Lobby zeigt alle Spiel-Metadaten
Die aktive Lobby (host_lobby.php) zeigt nur Teilnahme-Code + Teilnehmerliste. Es fehlen Fragenpool, Anzahl Fragen, Zeitlimit, Punkte-Modus. (Nur die statische Demo lobby.html zeigt sie hartkodiert.) QR-Code ist optional.
teilweiseSollLastenheft §10
Antwortreihenfolge-Fallback mischt erneut
Im Normalfall wird die gemischte Reihenfolge korrekt als JSON gespeichert (gut). Der Fallback-Pfad in game.php (Z.41-56, wenn quiz_data leer) macht jedoch ein erneutes shuffle() pro Spieler/Reload → unterschiedliche Reihenfolgen. Fallback angleichen.
teilweiseSollLastenheft §11.4
Startseite mit „Teilnehmen" / „Hosten"
index.html leitet sofort auf login.html um. Die beiden Optionen erscheinen erst im Dashboard nach Login/Gast-Auswahl. Wording „eröffnen" statt „hosten".
teilweiseSollLastenheft §7
Avatar beim Beitritt ist Pflicht statt optional
join_quiz.php (Z.32-34) erzwingt einen Avatar. Lastenheft sieht Avatar als optional vor. Kleiner Punkt – ggf. bewusste Teamentscheidung.
teilweiseKürLastenheft §8.1
Polling-Intervall ~1 s
Empfohlen 1 s; real 1,5 s (Spiel) bzw. 2 s (Lobby). Funktioniert, nur Feinheit.
teilweiseKürLastenheft §26.2
✔ Bereits umgesetzt: Host-Einstellungen (Pool/Anzahl/Zeit/Modus/Host-spielt-mit), Teilnahme-Code-Format (5 Zeichen, ohne 0/O/1/I), Beitritt nur in Lobby, Dozentenmodus, Auflösung + Erklärungen + Zwischenstand + „Nächste Frage" (Host), Polling-Mechanik
